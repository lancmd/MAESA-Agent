# 执行契约

## 默认行为

用户提供数据与目标后，智能体负责检查输入、选择后端、调用软件、验证产物和报告异常。能执行的步骤不得退化成软件菜单教程。源数据保持只读，所有派生文件进入任务工作目录。

## 阶段状态

| 状态 | 含义 |
|---|---|
| `completed` | 命令成功且声明的输出存在 |
| `prepared` | 已生成软件可直接读取的脚本或任务包 |
| `waiting_interactive` | 已到达必须登录、授权或 GUI 接管的位置 |
| `pending_validation` | 产物存在，但缺少独立样本或人工核验 |
| `failed` | 命令失败、必要输入缺失或输出验收失败 |
| `skipped` | 阶段未启用、依赖失败或已由状态文件确认完成 |

## 可恢复性

执行器把每个阶段的开始时间、结束时间、命令、返回码、日志与输出写入 `agent_state.json`。重复运行时，只有状态为 `completed` 且输出仍存在的阶段可以跳过；更改输入或参数后应使用新任务 ID 或删除该阶段状态。

## 阻塞边界

以下情况允许暂停对应阶段：必要数据未提供、商业软件许可不可用、云服务未认证、PLUS 版本没有已验证的自动化入口。不得用“建议用户操作”掩盖程序错误；不得将生成脚本描述为已生成模型结果。

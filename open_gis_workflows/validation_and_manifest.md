# 地理数据质量验证与可复现清单

本模块用于在 GEE、ENVI、ArcGIS、GDAL、PLUS 和 InVEST 之间建立统一的质量门槛。没有独立分类验证样本时，仍需执行技术质量验证；二者不能混为一谈。

## 1. 三类验证

| 类型 | 回答的问题 |
|---|---|
| 分类精度验证 | 地类判别是否正确；需要独立真值或人工验证 |
| 技术验证 | CRS、网格、NoData、数据类型和文件是否正确 |
| 过程验证 | 输入版本、参数、软件和输出能否重现 |

缺少独立验证样本时，可以继续处理，但必须将分类图标记为“待精度验证”；技术和过程验证仍是必做项。

## 2. 输入门槛

对每个输入检查：

- 文件可读且波段/图层数量符合预期；
- CRS 已声明并与实际坐标数值相符；
- 时间、单位、比例因子、NoData 和数据类型明确；
- 矢量几何有效，无大量空几何或重复记录；
- 数据许可允许当前分析和成果发布；
- 下载日期、产品 ID 或版本已固定。

任何缺少 CRS、单位或类别说明的关键输入都应暂停进入模型，先确认元数据。

## 3. 对齐后门槛

所有 PLUS/InVEST 输入与 master grid 对比：

- CRS 字符串与实际坐标一致；
- GeoTransform、Cell Size、Extent、Rows/Columns 完全一致；
- 分类栅格只含合法整数编码；
- 连续因子值域合理，没有插值过冲和未解释空洞；
- NoData 未被误写成有效 0；
- 人口/GDP 总量型栅格处理前后总量符合预期；
- 粗分辨率数据的原始有效尺度保留在说明中。

仅比较地图外观不能证明栅格对齐。

## 4. 常用检查命令

```powershell
# 栅格元数据和统计
gdalinfo -json -stats output.tif

# 矢量图层结构、CRS、范围和要素数
ogrinfo -al -so output.gpkg

# Windows SHA-256
Get-FileHash output.tif -Algorithm SHA256

# 可选：检查 COG 结构
rio cogeo validate output_cog.tif
```

命令不可用时，用 ArcGIS Pro Raster Properties、地理处理历史和文件哈希完成同等检查。

## 5. 数据清单

复制并填写 `templates/data_manifest.json`。每次运行至少记录：

- 项目和研究区 ID；
- 输入来源、产品/场景 ID、版本、日期、URL 和许可；
- 原始 CRS、分辨率、单位和 NoData；
- master grid 的 CRS、GeoTransform、范围和行列数；
- 软件版本及关键工具参数；
- 输出格式、波段、数据类型和 SHA-256；
- 已完成的技术验证与分类精度状态。

不要把 `latest` 当成可复现版本。STAC 固定 Item ID，OSM/政府门户记录提取日期，动态数据记录时间范围和聚合方式。

## 6. 许可与署名

- 保留 Sentinel/Copernicus、Landsat/USGS、OSM、WorldPop 和政府数据的来源说明；
- 公开地图、报告或可下载数据按来源条款添加署名；
- OSM 派生数据发布前检查 ODbL 义务；
- 不因格式转换删除来源、许可或时间字段；
- 涉及采矿权、工作面或敏感工程数据时，先确认共享范围。

## 7. 交付门槛

交付前确认工作目录不依赖临时内存图层，所有输出可重新打开，类别表与文件一致，清单路径有效。若某项验证未完成，在清单中写明 `pending` 和原因，不得用空白字段假装已通过。
